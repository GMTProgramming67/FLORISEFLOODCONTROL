Source Code

FloRise firmware.

Written in Arduino IDE. The program uses non-blocking millis()-based state machines — no blocking delay() calls. Obtains stable readings by taking five readings from the JSN-SR04T, averaging only valid readings to minimize sensor noise and ensure measurement accuracy. Full source on GitHub.

FloRise.ino
#include <Wire.h>
#include <Adafruit_GFX.h>
#include <Adafruit_SSD1306.h>
#include <NewPing.h>
#include <TimerFreeTone.h>

#define TRIG_PIN 9
#define ECHO_PIN 10
#define MAX_DISTANCE 600

// Optimal Time Interval: 15s (validated by One-Way ANOVA)
const unsigned long interval = 15000;
const int SAMPLES = 5;
const unsigned long SAMPLE_INTERVAL = 30;

// Non-blocking sampler state
unsigned long lastSampleMillis = 0;
unsigned int sampleTotal = 0;
int sampleCount = 0, validSamples = 0;
bool sampling = false, samplingForA = false;

// Non-blocking buzzer state
int buzzFreq, buzzCount, buzzDuration, buzzIndex;
bool buzzActive = false, buzzToneOn = false;
unsigned long buzzMillis = 0;

// Di = Initial Reading, Df = Final Reading
float distanceA = 0, distanceB = 0;
bool firstReading = true;

void setup() {
  Serial.begin(115200);
  startSampling(true);
}

void loop() {
  unsigned long now = millis();
  handleSampler(now);
  handleInterval(now);
  handleBuzzer(now);
}

void processReading() {
  // Rate of Rise = (Di - Df) / t
  float ror = (distanceA - distanceB) / (interval / 1000.0);
  // Estimated Time = Df / RoR
  float et = (ror > 0.1) ? distanceB / ror : 0;
  showOLED(distanceB, ror, et);
  // Alerts: Safe <30cm | Moderate 30-70cm | Risky 70-80cm | Danger >80cm
}

1

Wire the JSN-SR04T ultrasonic sensor

Connect JSN-SR04T: VCC → 5V, GND → GND, TRIG → Pin 9, ECHO → Pin 10. The waterproof design allows the sensor to be mounted outside the enclosure, pointed downward above the water channel. The JSN-SR04T emits sound waves and measures the time for them to return after bouncing off the water surface, allowing for precise distance measurement.

2

Connect the OLED display

The SSD1306 OLED uses I²C: VCC → 3.3V, GND → GND, SCL → A5, SDA → A4. It is used to display information such as the Rate of Rise, Estimated Time, current water level, and flood alert status. Confirm I²C address is 0x3C using an I²C scanner sketch before proceeding.

3

Attach the piezo buzzer

Connect the passive piezo buzzer: positive lead → Pin 8, negative → GND. The buzzer serves as the alarm system, triggering different tones across four flood alert conditions — Safe, Moderate, Risky, and Danger — based on the current distance reading of the JSN-SR04T.

4

Connect the MB102 power supply

The MB102 breadboard power module is used to power the whole circuit, including the Arduino. Connect it to a 9V DC source or USB. This ensures stable voltage delivery to all components during continuous flood monitoring operation.

5

Install Arduino libraries

Open Arduino IDE → Library Manager → install: Adafruit GFX, Adafruit SSD1306, NewPing, and TimerFreeTone. All are available via the Arduino Library Manager at no cost.

6

Upload, calibrate, and deploy

Upload FloRise.ino from the GitHub repository. Open Serial Monitor at 115200 baud to verify readings. The system takes five readings from the JSN-SR04T per interval, averaging only valid readings to minimize sensor noise. Mount above the flood channel and set the maxd threshold to match your site's danger level (default: 70 cm).

FloRise: Real-Time Flood Estimation System Using Ultrasonic Sensing and Arduino Technology via Rate of Rise Analysis.

Gian Markus C. Taguibao
Lead Researcher · Looc Integrated School
Aldrin Clarence N. Llamazares
Researcher · Looc Integrated School
Mrs. Janice Lee
Research Adviser

gianmarkustaguibao@gmail.com · aldrinllamazares6@gmail.com

Abstract

This study aims to design and develop a flood monitoring prototype capable of estimating real-time flood progression through Rate of Rise computation. The study starts with the researchers creating a formula that will measure the Rate of Rise, then creating the product along with the Estimated Time Formula. The system used an ultrasonic sensor (JSN-SR04T) along with an Arduino Uno Nano as its microcontroller, an OLED display, and a buzzer alarm to alert civilians. To assess the system, four different time intervals were tested across five trials each to determine the Optimal Time Interval for Rate of Rise measurement, with the Margin of Error (MOE) recorded for each trial and then averaged. Results showed that the system effectively measured water level changes and provided consistent, real-time predictions of flood progression with minimal error and low response delay. The findings indicate that FloRise is a reliable, low-cost, and efficient flood estimation device that can be used as an early warning system for communities that are often at risk of floods.

Keywords: Arduino, Flood Monitoring, Rate of Rise, Automated, Prediction, Estimated Time

Introduction

Flooding remains a significant natural hazard in the Philippines, a country frequently affected by typhoons and heavy rainfall. In 2020, the National Disaster Risk Reduction and Management Council (NDRRMC) reported that floods impacted over 1.5 million individuals nationwide. Flooding is frequent in urban areas, where heavy rainfall along with solid waste clogging canals prevents proper rainwater runoff. Traditional flood monitoring methods often depend on manual water gauges, which can result in delayed reports. Consequently, the need for real-time, automated flood systems has become increasingly important.

Floods pose serious dangers to life, property, and public health, often leading to damage of infrastructure and the spread of waterborne diseases. The Philippines has established legal frameworks to address these risks, notably Republic Act No. 10121, or the Philippine Disaster Risk Reduction and Management Act of 2010, which mandates the development of disaster preparedness and response systems including flood risk management and the utilization of scientific knowledge and technology in early warning systems.

This study aims to use ultrasonic sensing powered by Arduino Technology and utilize Rate of Rise in water levels to estimate the amount of time the water levels reach a certain distance. By utilizing Rate of Rise, the research provides real-time estimations and builds a flood progression system where citizens can be notified, potentially improving flood preparedness and reducing property damage while being cost efficient, affordable, and easy to produce.

Statement of the Problem

The study aims to design and develop a real-time flood progression estimation system using ultrasonic sensing and Arduino technology, utilizing the Rate of Rise to find the Estimated Time for peak flood arrival. Specifically, the researchers aimed to answer: (1) How accurate is the Rate of Rise? (2) How effective is FloRise in terms of ultrasonic sensor accuracy, optimal time interval, and measurement range? (3) How accurate is the Estimated Time for flood water to reach a set distance of 70 cm? (4) Is there a significant difference in the response time of FloRise when using different time intervals?

Hypothesis: There is no significant difference in the accuracy of the response time of FloRise when using different time intervals.

Methodology

The first phase utilized a Comparative Experimental Design to determine the most accurate time interval by testing four different intervals (5s, 10s, 15s, 20s) across five trials each, with the Margin of Error recorded for each trial and averaged. The second phase applied a Repeated Measures Design (RMD), where the most accurate interval was consistently used across all subsequent tests, with five trials conducted per test to ensure reliability and consistency of results.

This study was conducted at the researchers' houses in San Isidro Heights and Mabuhay Subdivision, Cabuyao, Laguna, and at Looc Integrated School. The researchers used the following formulas:

Rate of Rise (RoR): (Di − Df) / t = RoR — where Di = Initial Reading, Df = Final Reading, t = Time Interval.

Estimated Time (ET): Df / RoR = ET — the predicted time for flood water to reach the maximum set distance of 70 cm.

Results & Discussion

The Rate of Rise accuracy test showed a margin of error that decreased from 0.02 cm/s in Trial 1 to a consistent 0.01 cm/s from Trials 2 through 5. The ultrasonic sensor demonstrated measurement errors ranging from 0 cm to 0.75 cm at a fixed distance of 50 cm — consistent with previous studies reporting minimal error in ultrasonic sensor-based flood monitoring systems (Nunez, 2018; Syakur et al., 2023; Magnaye et al., 2024).

The 15-second interval yielded the fastest response time of 0.96 seconds among all tested intervals, as it provides enough time for the sensor to stabilize between readings. The JSN-SR04T successfully detected objects from 100 cm up to 600 cm, while detection failed at 700 cm, confirming that FloRise operates within the practical bounds of ultrasonic sensing for real-time water level estimation.

For Estimated Time prediction, the margin of error decreased progressively from 32.62 seconds in Trial 1 to 16.11 seconds in Trial 5, with an overall average margin of error of 24.14 seconds representing an error rate of approximately 6%. The One-Way ANOVA produced a Computed Value of 4,395.77 against a Tabular Value of 3.2389, resulting in rejection of the null hypothesis at α = 0.05.

Conclusion

The One-Way ANOVA test concluded that there is a significant difference in the accuracy of the response time of FloRise when using different time intervals. Since the CV (4,395.77) is greater than the TV (3.2389), the null hypothesis is rejected at α = 0.05 across the 5-second, 10-second, 15-second, and 20-second intervals, confirming that the 15-second interval is the most effective for measuring the Rate of Rise. It is short enough to capture meaningful changes in water level, but long enough to produce stable and accurate readings. FloRise is a reliable, low-cost, and efficient flood estimation device that can serve as an early warning system for communities at risk of floods.

Recommendations

Based on the findings, the researchers recommend: (1) Replacing the OLED display with a GSM Module to transmit flood data directly to mobile phones, and the use of an ESP32 microcontroller with built-in WiFi and Bluetooth for wireless data transmission. (2) Integrating a solar panel as a power source to ensure FloRise remains operational during power outages caused by flooding. (3) Future researchers are encouraged to conduct testing in actual flood-prone areas rather than a controlled environment to validate real-world accuracy, and to test FloRise under different weather conditions such as rain and wind to evaluate its outdoor performance.
